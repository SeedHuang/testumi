# DesginAble

首先是确定下来组件的制作方式，目前来说组件的建造方式目录结构，以及可视化配置部分很原始，需要有脚手架来进行编辑，另外，需要将 formilyi 的组件，和 antd 组件，以及自定义的组件进行整合，在目录上可以区分，但在实际应用上需要将其整合为一个整体，目前整合一个组件，需要在多个文件下面进行配置，并不是导入一个就是一个的状态，这个问题不解决，在后面会很麻烦，

应当是一个包就可以导入的，包括他的 schema，locales，组件本身而不是会有一个侵入式的在 designable 内部的配置进行的，designable 需要做的是，建立一个加单的加载机制，这样就可以把组件动态的加载进来了，关于这个分为两步走，一个是将组建隔离出去，另外一个是动态化加载；
